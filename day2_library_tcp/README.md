# Day 2 - 공유 라이브러리 + TCP 기본 구현

## 개요
Day 1에서 작성한 장치 제어 함수들을 각각 독립적인 공유 라이브러리(.so)로 분리하고
dlopen/dlsym을 활용한 동적 로딩 방식으로 TCP 서버/클라이언트 기본 틀을 구현했습니다.

## 라이브러리를 하나로 합치지 않은 이유

장치별 기능을 독립적인 공유 라이브러리로 분리하면 LED, 부저, 조도센서, 7세그먼트 중 특정 장치 기능만 수정할 때 
해당 라이브러리만 다시 빌드하여 교체할 수 있습니다.

서버는 dlopen/dlsym을 이용해 필요한 장치 제어 함수를 동적으로 로딩하므로, 
서버 코드와 장치 제어 코드를 분리할 수 있습니다. 
이를 통해 장치 기능 업그레이드 시 전체 서버 코드를 수정하지 않고 변경된 라이브러리만 교체할 수 있습니다.

## 파일 구성
| 파일 | 설명 |
|------|------|
| led.c | LED 제어 공유 라이브러리 소스 |
| buzzer.c | 부저 제어 공유 라이브러리 소스 |
| cds.c | 조도센서 공유 라이브러리 소스 |
| segment.c | 7세그먼트 공유 라이브러리 소스 |
| temperature.c | 온도센서 공유 라이브러리 소스 |
| test_device.c | dlopen/dlsym으로 전체 장치 동작 테스트 |
| server.c | epoll 기반 TCP 서버 기본 틀 (라즈베리파이 실행) |
| client.c | 메뉴 기반 TCP 클라이언트 기본 틀 (우분투 실행) |
| Makefile | 라이브러리/서버/클라이언트 빌드 자동화 |

## 아키텍처
클라이언트 (우분투)
→ TCP 명령 전송 (LED ON, BUZZER OFF 등)
→ 서버 (라즈베리파이)
→ dlopen/dlsym으로 동적 로딩
→ pthread로 명령마다 스레드 생성
→ 공유 라이브러리 (.so) 장치 제어

## test_device.c 테스트 내용
dlopen/dlsym으로 각 라이브러리를 동적으로 로딩하여 전체 장치 동작을 순서대로 테스트했습니다.

- LED ON/OFF 및 밝기(HIGH/MID/LOW) 동작 확인
- 부저 ON/OFF 및 음악 재생 확인
- 조도센서 값 5회 반복 읽기 확인
- 온도센서 값 5회 반복 읽기 및 온도 변환 확인
- 7세그먼트 0~9 순차 출력 확인

## 명령어 프로토콜
| 명령 | 동작 |
|------|------|
| LED ON/OFF | LED 켜기/끄기 |
| LED HIGH/MID/LOW | 밝기 조절 |
| BUZZER ON/OFF | 부저 켜기/끄기 |
| SEG [0-9] | 카운트다운 시작 |
| CDS | 조도센서 값 반환 |
| TEMP | 온도값 반환 |

## 빌드 및 실행

### 빌드 순서
1. 장치별 공유 라이브러리(.so) 빌드
2. TCP 서버 빌드
3. TCP 클라이언트 빌드
4. 서버 실행파일과 공유 라이브러리를 Raspberry Pi로 자동 전송

### 실행
```bash
# 전체 빌드 및 라즈베리파이 전송
make

# 라즈베리파이에서 서버 실행
sudo ./server

# 우분투에서 클라이언트 실행
./client \[라즈베리파이 IP\]
```
